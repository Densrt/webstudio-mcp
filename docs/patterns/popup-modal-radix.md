# Popup Modal pattern (Radix Dialog) — validated in production 2026-05-23

**Why**: a promo popup (newsletter, seasonal offer, exit-intent) requires, without a tool, ~12 instances + 50 styles + an HtmlEmbed JS script — repetitive and error-prone. This pattern hardens the construction in a single pass: mandatory a11y, managed frequency, Radix-compatible programmatic opening.

**How to apply**: for any Webstudio modal popup whose content is a promotional image (with or without a link). Prefer `build.create_popup`, which produces this structure out of the box.

## Architecture (validated)

**1 wrapper instance** under the provided parent (typically the page root), containing the Dialog AND the JS script:

```
wrapper Box (label "Popup" — idempotent replace target)
├─ Dialog
│   ├─ DialogTrigger
│   │   └─ <button id="..." aria-hidden tabIndex=-1>  ← hidden (off-screen + opacity:0)
│   └─ DialogOverlay (full-viewport, centers the content)
│       └─ DialogContent (centered white card, max-width 500px, radius 8px)
│           ├─ DialogTitle (sr-only — a11y)
│           ├─ DialogDescription (sr-only — a11y)
│           ├─ [Link?] > Image (assetId, alt, width/height)
│           └─ DialogClose
│               └─ <button>×  (top-right by default)
└─ HtmlEmbed <script>  ← auto-open + frequency check
```

## Critical rules (otherwise it breaks)

### 1. Programmatic opening = `.click()` on the hidden trigger button

Radix Dialog manages its `open` in React state — impossible to `setAttribute('data-state','open')` by hand. The only stable pattern:

- Keep a real `DialogTrigger > <button>`
- Make it invisible (off-screen + `opacity:0` + `pointerEvents:none`) AND inaccessible (`aria-hidden="true"`, `tabIndex={-1}`)
- Give it a stable HTML `id` (not the `ws-id`, which changes on every rebuild)
- The script calls `document.getElementById('<id>').click()` — Radix reacts as if to a real user click

```html
<!-- The HtmlEmbed generated by create_popup -->
<script>(function(){
  if(sessionStorage.getItem('popup_seen'))return;
  setTimeout(function(){
    var t=document.getElementById('abc-trigger-html');
    if(t){t.click();sessionStorage.setItem('popup_seen','1');}
  },2000);
})();</script>
```

### 2. Frequency = check before opening, mark after opening

Order matters: **read the flag right at the IIFE** (early exit if already seen), and **write the flag right after `.click()`** (not before — if the user has a bug and the popup doesn't open, we don't want to mark it as seen).

- `once-per-session` → `sessionStorage` (key `popup_seen` by default, writes `"1"`)
- `once-per-user` → `localStorage` with value `{ts: Date.now()}` and a TTL via `expiryDays` (default 30 days). Not a simple flag: otherwise it's impossible to "refresh" the campaign 3 months later without asking every visitor to clear their cache.
- `always` → no check, no mark (test/dev mode only)

### 3. `exit-intent` = desktop only (`hover: hover`) with a fallback timer

The document's `mouseleave` with `clientY <= 0` is reliable **desktop only**. On touch (mobile/tablet), there is no mouse → `mouseleave` never fires.

The script tests `window.matchMedia('(hover: hover)').matches` and falls back to a `setTimeout` (5000ms by default, override via `trigger.delayMs`) if the pointer is not hoverable.

### 4. HtmlEmbed with `<script>` = sibling of the Dialog, inside the wrapper Box

Important: the `<script>` does NOT go inside the `DialogContent` (otherwise it only runs when the Dialog opens — a contradiction: it's the one that should OPEN the Dialog). It goes in the wrapper Box, next to the Dialog, so it runs on page mount.

Also: `executeScriptOnCanvas: false` — otherwise the Webstudio builder tries to run the script in the canvas iframe, which pollutes the editor (popup opening during editing).

### 5. Native Radix `DialogClose` > custom close button

The `DialogClose > <button>×` wrapper provides for free:
- Esc to close
- Outside click (on the overlay) to close
- Focus management (returns focus to the trigger after close)

Do not implement a "manual" close button that mutates `data-state` itself — it breaks Radix focus management.

### 6. Mandatory a11y: DialogTitle + DialogDescription

Otherwise Radix logs on every open:
```
Warning: `DialogContent` requires a `DialogTitle` for the component to be accessible.
Warning: Missing `Description` or `aria-describedby` for `{DialogContent}`.
```

Both are injected sr-only (clipPath inset(50%) + position:absolute + 1×1px) — visually invisible but read by screen readers. See also: pattern `sheet-mobile-radix`, which applies the same sr-only block.

Overridable via `a11yTitle` / `a11yDescription` (`null` to opt out — **not recommended**).

## When to use

- Newsletter sign-up modal on Home after 2-5s
- Seasonal offer / promo (Black Friday, sales)
- Exit-intent retention (1 chance to keep a leaving visitor)
- Alternative cookie banner (in `manual` mode wired to a "Preferences" button)

## When NOT to use

- **Mobile drawer / burger nav** → `webstudio_create_sheet`
- **Desktop mega menu** → `webstudio_create_navigation_menu`
- **Free content inside the popup** (not an image — a form, cards, a product) → `webstudio_push_fragment` with a manual Dialog structure for now. The `kind: "custom"` is planned for v2 of the tool.
- **Toast notification** (non-blocking, auto-dismiss) → not a Dialog but a fixed `Box` component with a custom timer, out of scope for this tool.
